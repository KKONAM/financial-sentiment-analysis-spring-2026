\begin{abstract}
\small
Financial return prediction is difficult because short-horizon stock movements are noisy, nonstationary, and sensitive to both market dynamics and new information. This project studies whether daily sentiment features, combined with technical indicators, can support sequence models for forecasting future stock returns and three-way return direction. We construct a panel dataset for AAPL, AMZN, META, NVDA, and TSLA using daily market indicators and StockTwits-derived sentiment features. Because the available StockTwits messages end in late February or early March 2022, we restrict the final experiment to the period with sentiment coverage. The target is the future five-trading-day return, evaluated under a purged chronological split so target windows do not cross train-validation or validation-test boundaries. We first evaluate tuned LSTM and GRU models trained only to predict continuous five-day returns. We then extend them into multitask models that share the same LSTM or GRU sequence layer. These multitask models use two output heads: one for return regression and one for three-class direction prediction. In the return-only setting, sentiment-only LSTM obtains the lowest recurrent-model test error (MAE 0.0488, RMSE 0.0662) and strongest positive test correlation (0.0461). In the multitask setting, sentiment-only GRU obtains the highest direction-head accuracy (0.4218), sentiment-only LSTM obtains the strongest positive test correlation (0.0365), and combined-feature LSTM obtains the best macro F1 (0.2883). The multitask direction heads improve over thresholding return-only outputs, but the always-down baseline remains stronger on raw accuracy because the held-out period is negatively skewed. These findings suggest that sentiment features contain measurable but weak signal, while financial direction prediction remains highly sensitive to regime shift and class imbalance.
\end{abstract}
\vspace{1em}

\section{Introduction}

Predicting stock returns is a central problem in financial machine learning, but it is also a difficult one. Daily returns contain substantial noise, and predictive relationships can change quickly as market regimes shift. Traditional approaches often rely on price-derived technical indicators, such as momentum, relative strength, moving-average convergence divergence, and volatility measures. However, financial markets also respond to information outside the price series, including news, social media discussion, earnings narratives, and investor sentiment.

This project investigates whether combining market indicators with daily sentiment scores improves short-horizon stock prediction. We frame the task as a supervised sequence learning problem: given a rolling window of recent financial and sentiment features, the model predicts the future five-trading-day return for a stock. Because practical use also depends on whether a stock is expected to increase, decrease, or remain approximately flat, we additionally evaluate three-way directional prediction. We evaluate recurrent neural networks because LSTM and GRU models are designed to summarize temporal dependencies in ordered data.

The main contribution of this experiment is an end-to-end empirical comparison between tuned LSTM and GRU models on a shared dataset and split. Rather than reporting only classification accuracy, we evaluate both regression error and directional correctness. We first study recurrent models trained only to predict continuous returns, where direction is obtained by thresholding the predicted return. We then implement multitask LSTM and GRU architectures that share the same sequence-processing layer but jointly optimize return magnitude and a three-class direction head. This distinction is important because a model can achieve low return error by predicting values close to zero while still failing to provide useful directional information.

\section{Data and Features}

\subsection{Dataset}

The dataset contains daily observations for five large-cap technology stocks: AAPL, AMZN, META, NVDA, and TSLA. The StockTwits data consist of scraped raw messages, searched by ticker, with optional user-provided \textit{Bullish} and \textit{Bearish} labels. These labels are not complete annotations for every message; an unlabeled message is therefore treated as unknown sentiment rather than neutral sentiment.

The available StockTwits message coverage varies slightly by ticker: AAPL ends on February 27, 2022, AMZN and FB/META end on March 5, 2022, NVDA ends on February 28, 2022, and TSLA ends on March 1, 2022. To avoid evaluating sentiment models in a period with no real sentiment observations, the final experiment is restricted to January 2020 through February 2022. After technical-indicator warmup and the five-day future-return target construction, the modeled dataset covers January 23, 2020 through February 18, 2022 and contains 2,625 stock-day rows.

The data are split chronologically to avoid look-ahead leakage: observations through April 30, 2021 are assigned to training, observations from May 1, 2021 through August 31, 2021 are assigned to validation, and observations after August 31, 2021 are assigned to testing. Because the prediction target uses a future five-trading-day return, we purge samples whose target window crosses a split boundary. Since sequence length is tuned, the exact number of training sequences varies by model configuration; for a representative 20-day lookback window, the final sequence split contains 1,485 training sequences, 400 validation sequences, and 595 test sequences.

\subsection{Market Features}

For each stock-day, we include technical and market-derived features:

\begin{itemize}
    \item daily return,
    \item relative strength index (RSI),
    \item moving-average convergence divergence (MACD),
    \item Bollinger Band position,
    \item momentum,
    \item trading volume.
\end{itemize}

These features summarize recent price behavior, trend, overbought or oversold conditions, and market activity.

\subsection{Sentiment Features}

The sentiment feature set consists of daily aggregated sentiment statistics:

\begin{itemize}
    \item positive sentiment score,
    \item negative sentiment score,
    \item neutral or missing-sentiment fallback score,
    \item aggregate sentiment score,
    \item sentiment confidence,
    \item article count,
    \item message count.
\end{itemize}

For StockTwits rows with explicit \textit{Bullish} or \textit{Bearish} labels, sentiment scores are computed directly from those labels and averaged by ticker-day. The raw StockTwits data used here do not provide an explicit neutral label, so the neutral score is not an observed neutral-sentiment annotation. Instead, it is a constructed fallback used when no labeled bullish or bearish sentiment is available for a ticker-day. Unlabeled rows are excluded from the sentiment averages, but they are retained in a separate message-count feature. Thus, article count measures the number of labeled sentiment messages, while message count measures total StockTwits activity. These variables are aligned at the daily ticker level and merged with the market indicators. Technical indicators are kept in their raw constructed scale during feature building; all model input features are then standardized once using only the training-period statistics, and the same scaler is applied to validation and test data.

\section{Methodology}

\subsection{Prediction Target}

The prediction target is the future five-trading-day return:

\begin{equation}
y_t = \frac{P_{t+5} - P_t}{P_t},
\end{equation}

where $P_t$ is the adjusted stock price at day $t$. The target is treated as a continuous regression value. For directional evaluation, predicted and actual returns are mapped to three classes using a flat threshold of 0.005:

\begin{equation}
d(y) =
\begin{cases}
1 & \text{if } y > 0.005, \\
-1 & \text{if } y < -0.005, \\
0 & \text{otherwise.}
\end{cases}
\end{equation}

This separates meaningful positive and negative moves from near-flat outcomes.

\subsection{Sequence Construction}

For each ticker, observations are sorted chronologically and converted into rolling sequences. A sequence contains $T$ daily feature vectors ending at day $t$, where $T$ is tuned as a hyperparameter. The model outputs one scalar prediction for the future five-day return from day $t$ to day $t+5$. This corresponds to a realistic end-of-day setup in which the day-$t$ market and sentiment features are available before the prediction is made. To prevent label overlap across chronological splits, training and validation samples are purged if their target end date falls beyond the corresponding split boundary.

\subsection{Models}

Both neural models follow the same high-level structure. For a stock-day sequence $X_i \in \mathbb{R}^{T \times d}$, where $T$ is the lookback length and $d$ is the number of input features, the LSTM or GRU layer reads the sequence and produces hidden states $H_i = (h_1,\ldots,h_T)$. These hidden states are reduced to a fixed-dimensional representation $z_i$ using a tuned pooling rule. The return-only models pass $z_i$ through a feedforward head:

\begin{equation}
\hat{y}_i = f_{\theta}(z_i),
\end{equation}

where $\hat{y}_i$ is the predicted future five-day return. Directional labels are then obtained only after prediction, by thresholding the continuous return prediction.

The final experiments add multitask versions of both architectures. The multitask models keep the same LSTM or GRU sequence layer, but attach two prediction heads:

\begin{align}
\hat{y}_i &= f_{\theta}^{(r)}(z_i), \\
\hat{\mathbf{p}}_i &= \mathrm{softmax}(f_{\theta}^{(d)}(z_i)),
\end{align}

where $\hat{y}_i$ is the predicted continuous return and $\hat{\mathbf{p}}_i$ is a probability distribution over the three direction classes: down, flat, and up. The model is trained with a weighted sum of regression and direction losses:

\begin{equation}
\mathcal{L} = \mathcal{L}_{reg}(y_i,\hat{y}_i) + \lambda \mathcal{L}_{ce}(d_i,\hat{\mathbf{p}}_i),
\end{equation}

where $\lambda$ is tuned. This architecture makes stable or near-flat returns an explicit class rather than treating them as an after-the-fact thresholding artifact.

\textbf{LSTM.} The long short-term memory model is designed to address vanishing-gradient problems in recurrent networks by maintaining both a hidden state $h_t$ and a cell state $c_t$ \cite{hochreiter1997lstm}. Its input, forget, and output gates control how much new information is written, how much past state is retained, and how much memory is exposed to the next layer. This makes the LSTM a natural baseline for financial sequences, where a signal may depend on multi-week patterns such as trend persistence, volatility buildup, or sentiment accumulation. To make the comparison with GRU more controlled, the LSTM tuner uses the same non-recurrent search space as the GRU tuner: sequence length, hidden size, recurrent depth, pooling strategy, layer normalization, regression-head depth, activation function, dropout, optimizer, weight decay, batch size, loss function, target scaling, and gradient clipping.

\textbf{GRU.} The gated recurrent unit is a more compact gated architecture introduced in encoder-decoder recurrent networks \cite{cho2014gru}. It merges the separate LSTM memory and hidden states into a single hidden state and uses update and reset gates to decide how much historical information to preserve or overwrite. This reduces the number of parameters relative to an LSTM with comparable hidden size, which can be useful in this project because the chronological training set is small and noisy. The GRU uses the same fixed forecasting information set and matched tuning space as the LSTM, with the recurrent cell type as the primary architectural difference.

The two architectures therefore test different inductive biases. The LSTM has a larger explicit memory mechanism, while the GRU has fewer recurrent parameters per hidden unit. Both searches restrict recurrence to the forward time direction, avoiding bidirectional encoders so that the model structure mirrors the forecasting chronology. In the multitask experiments, the LSTM or GRU cell type remains the core architectural difference; the shared encoder and two-head output design are otherwise matched.

\subsection{Hyperparameter Selection}

Hyperparameters are selected using validation performance only. Each LSTM and GRU experiment uses a 60-trial random search with a fixed training seed for reproducibility. The return-only tuning objective combines RMSE and MAE with rewards for positive correlation and directional lift over the validation majority-class direction baseline, plus penalties for negative correlation and prediction collapse. The collapse penalty is included because early trials sometimes achieved competitive RMSE by predicting nearly constant small returns, which is not useful for directional forecasting.

The multitask tuner uses the same core architecture space while also tuning the direction-loss weight and whether to apply balanced class weights to the cross-entropy direction loss. The multitask selection score includes regression error, correlation, direction-head lift, and macro F1. Macro F1 is included because the flat class is rare and raw accuracy can be dominated by the majority direction. Candidate models use Adam or AdamW optimization \cite{kingma2015adam,loshchilov2019adamw}. Random seeds are recorded to make stochastic initialization and training reproducible, but they are not treated as model hyperparameters. The final checkpoint for each model is selected from the best validation epoch under early stopping.

\subsection{Feature Ablations}

To evaluate the contribution of each feature family, we run two return-only ablation settings for both LSTM and GRU models. The technical-only setting uses returns, RSI, MACD, Bollinger Band position, momentum, and volume. The sentiment-only setting uses the seven daily sentiment and activity features. For each subset model, we run a separate 60-trial random search over the same matched hyperparameter space and select the best validation checkpoint. This subset-specific tuning is more informative than simply removing columns from a previously tuned combined-feature model, because the optimal sequence length, recurrence width, optimizer, and regularization can change when the input representation changes.

Because the technical-only ablations were weaker than sentiment-only models in the return-only study, the multitask follow-up focuses on the two most relevant feature settings: combined features and sentiment-only features. We tune four multitask models in total: sentiment-only LSTM, sentiment-only GRU, combined-feature LSTM, and combined-feature GRU.

\section{Experiments}

\subsection{Evaluation Metrics}

The primary regression metrics are mean absolute error (MAE) and root mean squared error (RMSE). We also report Pearson correlation between actual and predicted returns, prediction mean, and prediction standard deviation. Directional performance is measured using the three-way direction mapping above. For return-only models, direction is computed by thresholding the predicted return. For multitask models, direction is computed from the three-class direction head. We report both accuracy and macro F1; macro F1 gives equal weight to down, flat, and up classes and is therefore more informative when one class dominates a chronological split.

\subsection{Baselines}

We compare the recurrent models against simple baselines:

\begin{itemize}
    \item zero-return prediction,
    \item training-set mean return prediction,
    \item always-up, always-down, and always-flat direction baselines.
\end{itemize}

These baselines are important because the September 2021 through February 2022 test period has a strong negative directional skew for several technology stocks. A naive always-down directional classifier is therefore unusually strong on the test split.

Under the matched return-only combined-feature search, the GRU obtains lower regression error than the LSTM, with MAE 0.0502 and RMSE 0.0680, but its test correlation is negative and its thresholded directional accuracy is only 0.3143. The combined-feature LSTM has MAE 0.0531, RMSE 0.0713, directional accuracy 0.3664, and correlation -0.0710. Both combined-feature recurrent models remain weaker than the zero-return baseline on regression error, and both underperform the always-down direction baseline in directional accuracy.

The feature-ablation results refine this conclusion. Sentiment-only LSTM achieves the lowest MAE and RMSE among recurrent models, with MAE 0.0488 and RMSE 0.0662, and it also has the strongest positive correlation at 0.0461. Sentiment-only GRU achieves the highest recurrent directional accuracy at 0.3899 and a similar positive correlation of 0.0459. Technical-only models do not improve the recurrent results on this split, suggesting that the StockTwits-derived sentiment and activity features carry more of the weak out-of-sample signal than the selected technical indicators.

The multitask models improve the directional behavior of the recurrent models, but only partially. For all four final feature-architecture pairs, the explicit direction head has higher test accuracy than the direction obtained by thresholding the return head. Sentiment-only GRU has the highest direction-head accuracy at 0.4218, followed by sentiment-only LSTM at 0.4185. Combined-feature LSTM has the best macro F1 at 0.2883, indicating that it is less dominated by one direction class than the other multitask models. However, none of the multitask models beats the always-down baseline's raw accuracy of 0.5025. This is mostly a class-distribution effect: the held-out period contains many more down examples than flat examples, so a naive always-down rule obtains high accuracy while having weak macro F1.

\section{Discussion}

The results show that sentiment-enhanced recurrent models can learn weak predictive structure, but the signal is limited and metric-dependent. After introducing the purged split, removing ticker-level normalization during technical-feature construction, fixing the forecasting information set, and matching the LSTM and GRU search spaces, the strongest return-only results come from sentiment-only models rather than the combined feature set. The sentiment-only LSTM has the lowest return-only recurrent-model error and the strongest positive correlation, while the sentiment-only GRU has the highest thresholded recurrent directional accuracy. This suggests that the StockTwits label and activity features contain weak out-of-sample information, but the signal is not strong enough to clearly beat simple baselines.

The multitask experiment directly addresses a weakness of the return-only setup. In the original models, direction was inferred by thresholding a scalar return prediction, so the model was never explicitly trained to distinguish down, flat, and up outcomes. The multitask architecture makes this distinction part of the model by training a direction head alongside the return head. This improves directional accuracy for all four final model settings. For example, sentiment-only GRU rises from 0.3899 thresholded directional accuracy in the return-only experiment to 0.4218 direction-head accuracy in the multitask experiment, while sentiment-only LSTM rises from 0.3647 to 0.4185.

However, the improvement is not enough to overcome the main difficulty of the held-out period. The test split is negatively skewed, so the always-down baseline reaches 0.5025 accuracy. The multitask models are more informative than this baseline by macro F1, but they still struggle to identify flat days and often collapse toward up or down predictions. This is visible in the confusion matrices saved with the model outputs: the sentiment-only multitask models assign almost no examples to the flat class. The flat class is therefore present in the task definition and the architecture, but it remains hard to learn from the available data.

The combined-feature results are also weaker than expected. Adding technical indicators to sentiment does not reliably improve test correlation or direction accuracy. One plausible explanation is that the selected technical indicators and raw volume features add noise or regime-specific relationships that are useful on the validation period but do not transfer to the September 2021 through February 2022 test period. This is consistent with the validation-test regime shift: the validation split is majority positive by direction, whereas the test split is majority negative. A model selected partly for validation directional lift can therefore lose directional accuracy when the market regime reverses.

Overall, the study argues against a simple conclusion that either architecture is uniformly better. The return-only LSTM is strongest on sentiment-only error and correlation, multitask GRU is strongest on direction-head accuracy, and multitask LSTM with combined features has the best macro F1. The more important finding is that all recurrent models remain fragile relative to simple baselines. The results should therefore be interpreted as evidence of weak sentiment signal and as evidence that multitask direction modeling is a useful refinement, not as evidence of a deployable forecasting model.

\section{Limitations}

There are several limitations. First, the dataset includes only five technology stocks, so the results may not generalize to broader sectors or market regimes. Second, the available StockTwits sentiment coverage ends around late February or early March 2022, limiting the amount of chronological test data available with real sentiment observations. Third, the user-provided StockTwits labels contain only explicit Bullish and Bearish tags; unlabeled messages are unknown rather than true neutral examples, and the neutral score is therefore a missing-sentiment fallback rather than an observed neutral label. Fourth, the sentiment features are daily aggregates, which may lose intraday timing information and message-level nuance. Fifth, although the multitask models explicitly include a flat direction class, the flat class is relatively rare and remains difficult to learn without stronger class-balancing or calibration. Sixth, the models optimize return regression and direction classification, but practical trading decisions may depend more directly on ranking, transaction costs, risk-adjusted returns, or portfolio-level objectives. Finally, the experiments use a finite 60-trial search budget and a single fixed training seed for reproducibility; a stronger robustness analysis would repeat the selected configurations across multiple training seeds and report the variance of the test metrics.

\section{Conclusion}

This study evaluates LSTM and GRU sequence models for five-day return prediction using combined technical and sentiment features. After correcting the sentiment coverage window, treating unlabeled StockTwits rows as unknown rather than neutral, using a purged chronological split, and matching the LSTM and GRU search spaces, the strongest return-only recurrent results come from sentiment-only ablations. Sentiment-only LSTM achieves the lowest return-only recurrent-model test error (MAE 0.0488, RMSE 0.0662) and the strongest positive correlation (0.0461), while sentiment-only GRU achieves the highest thresholded recurrent directional accuracy (0.3899).

The multitask extension improves directional prediction by making down, flat, and up movement explicit model outputs. Sentiment-only GRU achieves the highest multitask direction-head accuracy (0.4218), sentiment-only LSTM keeps the strongest positive multitask correlation (0.0365), and combined-feature LSTM achieves the best multitask macro F1 (0.2883). These gains show that the multitask architecture is a better match for the financial decision problem than training only on continuous return prediction, but the models still do not clearly outperform simple baselines across all metrics. Future work should consider stronger class balancing, calibrated decision thresholds, richer textual representations, broader cross-sectional data, and objectives tied more directly to ranking or risk-adjusted trading performance.
